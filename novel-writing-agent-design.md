# 写小说智能体产品与技术设计

## 1. 产品定位

这是一个面向小说作者的创作型 AI Agent。它不只是聊天助手，而是一个「创作工作台 + 长篇记忆系统 + 协作型编辑」。

它可以帮助作者完成从灵感、设定、提纲、分章、正文、润色、续写、改稿到发布整理的完整流程。

核心价值：

- 长篇小说上下文管理
- 世界观、人物、剧情线一致性维护
- 多风格写作与改写
- 分章节创作和版本管理
- 多平台同步创作
- 作者主导，AI 辅助

## 2. 目标用户

### 2.1 网文作者

需要高频更新、章节续写、爽点节奏、人物关系维护、平台发布格式整理。

### 2.2 长篇小说作者

需要复杂设定、伏笔回收、多线叙事、长期创作记忆。

### 2.3 剧本与游戏剧情作者

需要角色弧光、对白、分支剧情、世界观设定管理。

### 2.4 轻度创作者

需要灵感激发、故事大纲、短篇生成、改写润色。

## 3. 核心功能

### 3.1 项目工作区

每本小说是一个独立项目，包含：

- 小说简介
- 类型标签
- 目标读者
- 文风设定
- 世界观设定
- 人物卡
- 地点卡
- 物品、势力、规则卡
- 主线剧情
- 支线剧情
- 章节列表
- 伏笔池
- 待办事项
- 草稿与版本历史

用户打开项目后，AI 自动加载该小说相关上下文，而不是每次从零开始。

### 3.2 小说记忆系统

记忆系统是产品的核心能力。

记忆分层：

- 全局记忆：作者偏好、常用文风、禁忌、常写题材
- 项目记忆：世界观、人物、故事基调、叙事规则
- 章节记忆：每章发生的事件、人物状态变化、留下的伏笔
- 语义记忆：向量检索，用于找相似设定、旧剧情、人物关系
- 结构化记忆：数据库字段，用于精确维护年龄、身份、能力、关系、时间线

示例问题：

> 上一章女主是不是已经知道男主身份了？

Agent 应该能检索章节摘要、人物状态和事件记录，给出明确回答。

### 3.3 创作模式

建议提供多个模式，而不是一个万能输入框：

- 灵感模式：生成题材、脑洞、反转、冲突
- 设定模式：创建世界观、角色、势力、规则
- 大纲模式：生成三幕式、卷纲、章节纲
- 正文模式：根据章节目标生成正文
- 续写模式：接着当前文本自然续写
- 润色模式：提升语言、节奏、氛围、对白
- 改写模式：改成爽文、悬疑、古风、轻小说等风格
- 编辑模式：检查逻辑漏洞、节奏问题、人物崩坏
- 复盘模式：总结本章、更新人物状态、记录伏笔

## 4. Agent 工作流

每次写作可以走一个标准流程：

1. 理解当前任务  
   判断用户想写设定、章节、对白、修改，还是检查问题。

2. 加载上下文  
   自动读取项目设定、相关人物、上一章摘要、未回收伏笔。

3. 制定创作计划  
   给出本次写作目标、冲突、情绪变化、信息释放点。

4. 生成内容  
   输出正文、提纲、设定或修改建议。

5. 自检  
   检查是否违背设定、人物动机是否合理、是否重复啰嗦。

6. 写入记忆  
   用户确认后，更新章节摘要、人物状态、伏笔和时间线。

## 5. 多平台客户端设计

建议采用「云端 Agent + 多客户端」架构。

客户端形态：

- Web：主力创作工作台，适合长文本编辑
- iOS / Android：移动端灵感记录、轻量续写、碎片化修改
- macOS / Windows：桌面写作体验，支持本地文件、快捷键、离线草稿
- 浏览器插件：辅助在小说平台发布、整理格式、读取评论反馈
- 微信 / Telegram / Discord Bot：快速记录灵感、随手问设定

不同客户端共享同一个后端项目数据。

## 6. Web 端信息架构

主界面可以类似专业写作软件。

左侧：

- 项目列表
- 章节树
- 设定库
- 人物库
- 伏笔池
- 时间线

中间：

- Markdown / 富文本编辑器
- 当前章节正文
- 版本对比
- AI 插入建议

右侧：

- AI 助手面板
- 当前章节目标
- 相关人物
- 设定引用
- 逻辑检查结果
- 快捷操作按钮

底部或浮层：

- 字数统计
- 更新目标
- 情绪曲线
- 章节节奏评分

## 7. 技术架构

推荐架构：

```text
Clients
  Web / iOS / Android / Desktop / Bot
        |
API Gateway
        |
Agent Orchestrator
        |
------------------------------------------------
| LLM Service | Memory Service | Project Service |
| Tool System | Search Service | Export Service  |
------------------------------------------------
        |
Database + Vector DB + Object Storage
```

核心模块：

- API Gateway：认证、限流、路由
- Agent Orchestrator：调度模型、工具、记忆和写作流程
- Project Service：小说项目、章节、人物、设定管理
- Memory Service：长期记忆、摘要、状态更新
- Vector Search：语义检索旧剧情和设定
- Consistency Checker：检查设定冲突、人物状态冲突
- Export Service：导出 TXT、Markdown、DOCX、EPUB
- Sync Service：多端实时同步

## 9. Agent 能力设计

可以拆成多个子 Agent。

### 9.1 主创 Agent

负责正文生成、续写、风格控制。

### 9.2 大纲 Agent

负责故事结构、章节规划、冲突设计。

### 9.3 设定 Agent

负责世界观、人物、规则、势力。

### 9.4 编辑 Agent

负责逻辑漏洞、节奏、重复、人物崩坏检查。

### 9.5 记忆 Agent

负责摘要、状态更新、伏笔登记。

### 9.6 风格 Agent

负责模仿用户文风、统一语气。

### 9.7 发布 Agent

负责格式整理、标题生成、简介、平台文案。

## 10. MVP 范围

第一版不要做太大。建议 MVP 包含：

- Web 客户端
- 用户登录
- 创建小说项目
- 章节编辑器
- 人物卡 / 世界观卡
- AI 生成章节大纲
- AI 续写正文
- AI 润色文本
- 自动章节摘要
- 基础长期记忆检索
- 导出 Markdown / TXT

暂时不做：

- 多人协作
- 复杂桌面端
- 自动发布
- 评论分析
- 完整离线模式
- 极复杂的 Agent 编排

## 12. 商业化方向

可以按创作者需求分层：

- Free：少量项目、有限 AI 次数
- Pro：长篇记忆、多项目、高级润色
- Studio：多人协作、剧本模式、版权资料库
- API / 私有部署：工作室、网文团队、游戏公司

可加值功能：

- 高级文风训练
- 长篇一致性检查
- 章节质量评分
- 平台发布助手
- 作品数据分析
- IP 世界观资产管理

## 13. 一句话总结

这个产品的核心不是「让 AI 写小说」，而是做一个懂这本小说的长期创作伙伴：它记得人物、世界观、伏笔和文风，能陪作者从灵感写到完本，并在 Web、移动端、桌面端之间无缝同步。
